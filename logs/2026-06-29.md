# 2026-06-29

## Land minigame v1 + full design

Designed Land with Robert (full design in `references/land-minigame.md`) and built
a capped v1. Land is the layered ecosystem minigame (vs `life`'s abstract Conway).

Design highlights:
- **Layered cells**: terrain / micro / plant / animal / other, coexisting (a
  pyramid/food web), one occupant per layer. The layers *are* the item-model
  taxonomy classes, so insertion routes by `species.class()` automatically.
- **Life → Land pipeline**: no spontaneous generation — you seed Land by bringing
  organisms (Archaea harvested from `life`) and inserting them. The `other` layer
  is a catch-all (toss any item in).
- **Leveling** (full vision): by species *established*, with population bars that
  rise with complexity (1 archaea, several bacteria, many plants…) → rewards a
  deep diverse ecosystem. Grid grows on the shared 1×1/2×1/2×2 formula.
- **"other" layer** hooks Land into mana/runes later (monoliths, spells).

v1 (built, capped low — content is thin: mud/water terrain, archaea):
- `LandCell { terrain, micro, plant, animal, other }`; `cells: Vec<Vec<LandCell>>`.
- Grid via the shared `_blocks_per_row`/`_blocks_per_column` (deleted the broken
  `4*level`); one inset sprite per cell.
- `ingest_item` routes by class onto a random cell (a `place(..., Layer)` helper):
  energy→pool, bulk→terrain, organism→its class layer, else→other; ejects the
  remainder.
- `evolve_fixed_update` mirrors life (energy + `EVOLVE_TICKS` cooldown, peek-then-
  mutate); the rule is archaea-only: dies off water, spreads to a water+empty
  neighbor (snapshot-then-step). No spontaneous generation.
- Leveling: `distinct_complexity()` (distinct ItemTypes across all layers), capped
  at `MAX_LEVEL=6`; bumps `max_achieved_complexity` + inserts `LevelingUp`.
  `levelup()` preserves cells into the larger grid.
- `cell_update`: click → `extract_top` removes the highest non-terrain layer
  (other→animal→plant→micro), ejects it; terrain stays.
- `render_cells`: repaints each cell to its `top()` layer's texture every
  FixedUpdate. Wired into main.rs; `#![allow(warnings)]` removed.

Built by a briefed subagent (spec = the design doc's v1 section, mirroring life),
then reviewed by hand and verified: `nix develop -c cargo test` 50/50 (7 new land
tests — routing, archaea evolve, extraction), clippy clean. Only `land.rs` +
`main.rs` changed (life.rs untouched).

**Known issue surfaced (filed under Bugs):** most items still `panic!` in
`draw()`, so tossing a non-renderable item into land (bulk → terrain → repaint)
crashes — same systemic gap that already affects chest slots / loose items. Fix:
make `draw()`/`palette()` fall back to a placeholder instead of panicking.

Not yet verified in the running game. Also resolved the land half of the "no tint"
TODO (cells use `Color::WHITE`).

## Land: ECS integration tests (automated stand-in for a manual playthrough)

Robert wanted assurance Land works without the slow manual grind of accumulating
items. Added two integration tests that drive the *actual systems* through a real
Bevy `World` (headless — no GPU/window/plugins), using `World::run_system_once`:
- `evolve_and_render_run_through_the_ecs`: spawns a Land minigame + child Cell
  entities, fuels it, runs `evolve_fixed_update` past the cooldown (asserts a step
  ran via the ECS — energy consumed), then runs `render_cells` and asserts it
  paints cell sprites *without panicking*.
- `click_extracts_top_layer_through_the_ecs`: sets `MouseState` over a cell, runs
  `cell_update`, asserts the top layer was cleared and a loose `Item` was ejected.

These cover the wiring (systems run, queries match the spawned entities), the
evolve/render/extract loop, and render-no-panic — most of what a manual land test
checks. They deliberately use only renderable content (mud/water/archaea/apple).
Not covered (inherently hard headless): on-screen appearance, real input, the
physics-collision → `ingest_item` path, the levelup respawn handoff, and balance.
52 tests pass, clippy clean. Same pattern can be applied to `life` if wanted.
