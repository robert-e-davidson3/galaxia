# 2026-06-20

## Adopted Sorcida's coordination structure

Ported the project-agnostic coordination layer from `~/projects/shared/sorcida` into this repo — the directory convention, not anything Sorcida-specific. Goal: the same way of working (history in `logs/`, settled facts in `references/`, procedures in `skills/`, forward work in `tasks/`) so context lives in versioned files instead of being re-derived each session.

### What was created

- **`logs/`** — append-only daily history (this file is the first). One file per day, `YYYY-MM-DD.md`: what changed, what was researched, and the _why_.
- **`references/`** — settled facts, one narrative per topic. Seeded by migrating the fat `CLAUDE.md`:
  - `repo-layout.md` ← the old **Architecture** section (core structure, key systems, core libraries, game flow).
  - `local-dev.md` ← **Development Environment** + **Common Commands** (Nix shell, `cargo` commands; noted that both build profiles use `opt-level = 3`).
  - `tech-stack.md` ← **Key Dependencies**, expanded against the real `Cargo.toml` (Bevy 0.14.1, `bevy_rapier2d` / `rapier2d`, the grid/array crates, `wyrand`, the noise + image crates, `serde`).
  - `code-style.md` ← the **Code Style** section (80-col `rustfmt`, ECS patterns, the consistent minigame interface).
- **`skills/`** — runnable procedures. Seeded `add-minigame.md` from the old **Adding New Minigames** section, rewritten in Sorcida's skill shape (When / Steps / Smell tests) and pointed at the real files (`minigame.rs`, `main.rs`, `setup_minigame_unlocks()`).
- **`tasks/board.md`** — forward-looking kanban (Now / Next / Backlog / Decisions pending), migrated from `TASKS.md`. Removed `TASKS.md`. Kept a banner noting the migrated `file.rs:line` references predate recent item/minigame work and need re-validation (the old list was "Last Updated 2025-06-26" and several recent commits touched items and minigame spawning) — I did **not** verify each TODO against current source, only reshaped the list.

### CLAUDE.md slimmed

Rewrote `CLAUDE.md` from a fat doc that inlined architecture/deps/style into a thin pointer, matching Sorcida's: a short overview plus **Directory layout** and **How to work here** sections that route to the four coordination dirs. The detailed content now lives in `references/` and `skills/` rather than being duplicated in `CLAUDE.md`.

### Why this shape

From Sorcida's `CLAUDE.md`: logs are append-only history (nuance over time), references are the current settled truth (promote a fact out of a log once it stops moving), skills are procedures written so an AI can run them cold, and the board is forward-only (finished work goes to the log, not a "Done" column). The split keeps `CLAUDE.md` small and the durable facts findable.
