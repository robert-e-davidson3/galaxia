### The crash (panic: "Invalid form Apple", item.rs identifier())

Leveling the ocean spawned tree for the first time; `tree::fixed_update`
immediately spawns an Apple fruit, and `UnpickedFruitBundle::new` calls
`Item::asset()` → `identifier()`, which had no arm for `PhysicalForm::Apple`
(only Gas/Liquid/.../Archaea) → `panic!("Invalid form Apple")`. Tree had never
spawned before, so this gap (and two siblings) were dormant. Fixed all three:
1. `identifier()` noun match: added Apple/Lemon/Lime ("specific life" forms).
2. `draw()` Apple loaded `assets/physical/apple.png` but the file is `Apple.png`
   — case mismatch that would panic in `load_image().unwrap()` on *picking* a
   fruit (Linux is case-sensitive). Fixed the case.
3. `asset()` returned `physical/Fruit.png` (no such file) for any fruit — fruit
   is textured by its *form*, not material, so it now uses the noun →
   `physical/Apple.png` (matches the asset). Unpicked fruit on the tree now
   resolves instead of showing a missing-texture box.

Added a regression test (`apple_fruit_identifier_and_asset_resolve`) in a new
top-level `tests` module in item.rs. 34 tests pass, build clean. Not yet
verified in-game — needs: level ocean → tree appears at (-350,250) → apples grow
→ pick one without crashing.

## Tree polish: scatter apples on the canopy + make fruit chest-storable

In-game verification of tree surfaced two issues (the "minigames flashing" was
just expected unlock-spawn, no bug):

- **Apples all grew at the tree center and overlapped.** `tree::fixed_update`
  spawned every fruit at local `(0,0,0)`. Now it places each new fruit at a
  random point in the leafy crown (constants `CANOPY_MIN`/`CANOPY_MAX`, derived
  by eyeballing the 300x300 oak sprite — upper-center, off the trunk) and rejects
  candidates within `FRUIT_SPACING` of existing fruit (queried via
  `UnpickedFruit` filtered to this tree). Best-effort: 24 tries then place
  anyway, since the canopy can fill. `FRUIT_RADIUS` (8.0) is now shared with the
  fruit's `CircularArea`. Added `ResMut<Random>` + a fruit `Query` to the system
  (Random was already an inserted resource; no main.rs change).
- **Apples weren't storable in the chest.** `ChestMinigame::can_accept` only
  allows solid forms (Lump/Block/Ball) until level 20+. Added an early return so
  any fruit form (Apple/Lemon/Lime) is always storable. Regression test
  `chest_accepts_fruit_at_level_zero`.

35 tests pass, clippy clean. Canopy region is a hand-estimate — may need nudging
in-game so apples sit on branches rather than floating in leaves.

## Designed the item model: packed identity + taxonomy

Long design conversation about the item/resource classification system, prompted
by the Apple crash and "fruits are solids — how does classification work?".
Findings and the agreed target are written up in `references/item-model.md`.

Key points:
- Current `PhysicalForm`/`PhysicalMaterial` conflate multiple axes (matter-state +
  shape + open identity; substance + life-stage). `#[repr(u8)]` caps forms at 256;
  the numeric reprs are speculative (no casts/serde) — real identity is the string
  `uid()` + `discriminant`.
- Agreed model: a `u64` **tagged, nested** id read MSB→LSB, where each tag gates
  the meaning of later bits. Closed taxonomic axes stay enums/fields; the open
  species axis becomes a 20-bit data-registry leaf. Taxonomy (single-parent) is
  prefix-encoded and prefix-maskable; cross-cutting facets (edible/flammable/…)
  live in a derived `Flags` bitset, not in identity.
- Bit budget (Robert's calls): domain 3; abstract kind 13 (16 w/ domain, 48 left);
  minigame which 13 (>100 planned); species 20 (~1M); physical structure 5; mana
  element 3 + intent 2 + subkind 8; energy a 6-bit kinds mask.
- Decisions (final — see the doc): Fruit is a **category** under
  Discrete·Inanimate, with a class-dependent `state` field (7-bit **freshness**
  0–127 for fruit, averaged Wurm-style by storage later); shape and
  processing/refinement are **separate** Bulk axes
  (Ore=raw vs Refined; shape incl. Gravel); added a quality/grade Bulk axis;
  cosmetic variant + amount are instance data (persisted, not class); domain `111`
  is an escape hatch for non-taxonomic items; substance/life classes are
  derivable-but-kept-for-masking; validity separate; reclassification implies a
  save migration (accepted, post-release).

Captured as a board task under Tech debt / architecture; includes the follow-up
to replace the chest `can_accept` fruit early-return with `is_solid()` once it
lands. Not started — it's a big change and not urgent; finishing minigames
continues meanwhile.

## Implemented the item model (wholesale refactor)

Realized `references/item-model.md` in code — full replacement of the flat
`PhysicalForm`/`PhysicalMaterial` enums, no bridge (Robert: "just replace
everything", typed not raw-u64). ~1250 lines changed across item.rs + 8 consumers.

- `PhysicalItem` is now `enum { Bulk(BulkItem), Discrete(DiscreteItem) }`.
  `BulkItem { structure, substance, processing, shape, quality }`,
  `DiscreteItem { species, state }`, `State { Stage(LifeStage), Freshness(u8), None }`.
  New closed-axis enums: Structure, Substance, SubstanceClass, Processing,
  BulkShape, LifeStage, Species, DiscreteClass, Animacy. Classes are derived
  methods (`Substance::class`, `Species::class`, `DiscreteClass::animacy`), not
  stored.
- `ItemType::pack()/unpack()` implement the doc's u64 layout (explicit
  variant→int maps, not `as u8`); derived class/animacy prefix bits written on
  pack and validated on unpack. Round-trip tests for all worked examples.
- Helpers `is_solid()/is_fruit()/freshness()`; `Item` constructors
  `solid/ore/liquid/powder/terrain/fruit/organism` (non-solid structures
  normalize shape/processing so identity ignores irrelevant fields).
- Consumers remapped: chest `can_accept` via `is_fruit`/`is_solid`; ball_breaker
  keyed on `Substance`; foundry ore = Raw bulk-solid → smelt to liquid;
  primordial_ocean/land/tree via the new constructors; inventory tests rebuilt.
- **Naming:** the bulk geometry enum is `BulkShape`, not `Shape`, to avoid
  colliding with `bevy_prototype_lyon::prelude::Shape` (every minigame globs both;
  the collision would recur otherwise). Renamed from the subagent's initial
  `Shape` after it had to fully-qualify it in 4 files.

Done via a briefed subagent (the type design + per-consumer mapping), then
reviewed: verified the new types by hand, ran `nix develop -c cargo test` (38
pass, up from 35 — added pack/unpack round-trips) and clippy (clean). Trusted the
real 1.96 compiler over rust-analyzer's stale-1.87 false errors. NOT yet verified
in the running game — the draw/identifier paths and minigame item flows (chest
storage, ball_breaker, foundry smelting, tree fruit) want an in-game pass.

## Item model: review cleanups (pack/unpack)

Post-review tidy of the pack/unpack code:
- Unified the `ItemType::pack()` dispatch binding name (`inner` for every arm) —
  the varied `p`/`m`/`e`/`a` names read as if meaningful.
- Dropped the redundant `& 0b111` on the domain extraction (`packed >> 61` on a
  u64 already leaves only the top 3 bits; the mask is only needed for the middle
  fields).
- Replaced the 14 hand-written `pack_*`/`unpack_*` enum<->int functions (~190
  lines) with `int_enum::IntEnum` derives + explicit discriminants on the 8
  enums (Substance, SubstanceClass, Processing, BulkShape, LifeStage, Species,
  DiscreteClass, Animacy) — same pattern `Rune` already used. pack = `x as u64`,
  unpack = `Enum::try_from(byte).ok()`; the mapping now lives once, in the
  discriminants (values byte-identical to before, round-trip tests confirm).
  Added a `u8::try_from` guard on the 20-bit species field so an out-of-range
  value fails instead of truncating into a valid species. `Structure` left inline
  (its encoding skips a slot for the Discrete discriminator). 38 tests, clippy clean.
