# 2026-06-28

## Item model: drop `Terrain` structure, rename `Structure` → `BulkStructure`

Two review-driven changes to the physical taxonomy (see `references/item-model.md`):

- **Renamed `Structure` → `BulkStructure`.** "Structure" is too generic a name for
  what is specifically the matter-state of a *bulk* material.
- **Removed the `Terrain` variant.** Terrain isn't a state of matter — a terrain
  tile is just its underlying material (solid mud, liquid water). The land
  minigame already stored ingested solids/liquids as terrain cells; only the
  *default* terrain used `Terrain`. Now the default is `Item::solid(Mud, Lump)`,
  unifying it with ingested terrain, and "this cell is terrain" is purely the land
  minigame's concern (its `terrain` grid holds ordinary material items;
  `terrain_is_water` already keyed on the substance, not the structure).
  Dropped `Item::terrain`, `STRUCT_TERRAIN`, and the Terrain arms in
  draw/identifier/pack/unpack. `BulkStructure` is now `{ Gas, Liquid, Powder,
  Solid }`. (The Bulk/Discrete distinction's encoding was reworked immediately
  after — see the next entry.)

No save-format concern (pack/unpack isn't wired to disk yet, pre-release). 38
tests pass, clippy clean. Not re-verified in-game (land is still dormant; the
visible consumers — chest/ball_breaker/foundry/tree — are unaffected by the
terrain change, but a fresh in-game pass never hurts).
## Item model: nest the bit encoding to match the types

Follow-up to the BulkStructure rename: the packed encoding didn't mirror the Rust
types. The types nest (`PhysicalItem::Bulk(BulkItem { structure, .. })` vs
`PhysicalItem::Discrete`), but the bits flattened it — a single 5-bit `structure`
field held Gas/Liquid/Powder/Solid AND `Discrete` as a sentinel value (the doc's
original layout). Confusing.

Re-laid the physical payload to nest like the types:
- `kind` tag [60:59] = Bulk | Discrete (the `PhysicalItem` variant).
- For Bulk, the matter-state is a nested `BulkStructure` field [58:56].

With `Discrete` no longer occupying a slot in the structure field, `BulkStructure`
is now a clean 4-variant enum, so it also got the `IntEnum` treatment (Gas=0…
Solid=3) — `pack` = `as u64`, `unpack` = `try_from`. Removed the `STRUCT_*` consts
(replaced by `PHYS_KIND_BULK`/`PHYS_KIND_DISCRETE`). Updated the doc's layout
tables and property-query descriptions (`kind=Bulk`/`kind=Discrete`).

No save-format concern (not wired to disk yet). 38 tests pass (round-trips
confirm the nested encoding), clippy clean.

## Implemented the life minigame (Conway's Game of Life)

life was fully defined but dormant (systems unregistered, not in unlocks, evolve
a `return;` stub, rendering never built). Wired it up end to end:

- **Conway:** `LifeMinigame::step()` (+ `next_cell` helper) — live cell survives
  on 2-3 neighbors, dead cell born on exactly 3 (inheriting a neighbor's species),
  no edge wraparound. Built functionally (`(0..h).map(...).collect()`) to avoid a
  needless-range-loop.
- **Economy:** `evolve_fixed_update` steps at most once per `EVOLVE_INTERVAL`
  (1s) and only when `energy >= 1`, consuming 1 energy per step. Peeks immutably
  (`&*minigame`) so it doesn't mark Changed unless it actually steps. Added a
  `last_evolve_time` field.
- **Ingestion:** `ingest_item(rand, item)` — energy items add to the energy pool;
  any other item seeds a random empty cell with a life form (Archaea). Updated the
  dispatch in minigame.rs to pass `rand`.
- **Rendering:** new `render_cells` system repaints every cell from the model each
  FixedUpdate (grids are tiny) — alive cells draw their life form via a cached
  texture (`cell_texture`, mirrors inventory's get_texture), dead cells go
  transparent. This is the first time `CellBundle::turn_on` is actually called.
- **Wiring:** registered `life::cell_update` (Update) and
  `(life::evolve_fixed_update, life::render_cells).chain()` (FixedUpdate); added
  `life` to the unlocks table (Primordial Ocean L1, like land/tree). Also fixed
  `cell_update`'s bogus `&mut Query<...>` param (would've failed to register as a
  system) and removed `#![allow(warnings)]` from life.rs (builds clean without it).
- Used Archaea as the life form because it's one of the few species that actually
  renders (`archaea_palette().draw_lump`); seeding/births are all Archaea for now,
  which keeps it panic-safe. The cell model is `Option<ItemType>` so it can hold
  varied life later once more species render.

Tests: blinker oscillator (period-2), lone-cell underpopulation death, and a
seed-fills-a-cell test. 41 pass, clippy clean. **Not yet verified in-game** —
needs: level Primordial Ocean to unlock life, drop items to seed cells, feed
**energy** items to make it evolve (needs an energy source), click cells to
harvest. Evolution is unobservable without an energy supply, so that part may not
be exercisable yet depending on what produces energy items.

## Life leveling: XP = Σ|births − deaths| (slice 1) + full design

Designed Life's progression with Robert and built the first slice. Full design
captured in `references/life-minigame.md`; highlights and reasoning here.

The core insight (Robert's): **XP = per-step |births − deaths|, summed.** This is
the criterion that isn't gameable — oscillators/still-lifes/gliders all net zero
(`|2−2|`, `|0−0|`) so the "free forever" patterns pay nothing; only genuine
population *change* scores. It also makes the **1×1 start viable** (a lone cell
dies → `|0−1| = 1` XP → level 1), which Robert wanted to keep. And the difficulty
curve falls out of one symmetric formula because the economy is asymmetric: deaths
are a bounded well (kill only what you seeded, seeding is random+slow) while an
engineered net-birth surplus is unbounded but impossible in vanilla Conway — so
late levels *force* the rule-bender items (extractor, etc.).

Slice 1 implemented:
- `step()` now returns `|births − deaths|` for the step (counts via zip over
  old/new grids).
- `evolve_fixed_update` accumulates `xp` and inserts `LevelingUp` when
  `level_by_xp(xp) > level` — the previously-missing levelup trigger. The generic
  `minigame::levelup` respawns at the larger grid.
- Leveling driver switched from `extracted` to `xp` (geometric: level N at
  `xp >= 2^(N-1)`, reusing the old log2 formula). Removed the `extracted` field;
  harvesting (cell_update) no longer credits XP — it's a payout, XP is births/deaths.
- Tests: step scores |b−d| (lone cell → 1, blinker → 0); `level_by_xp` geometric.
  43 pass, clippy clean.

Roadmap (slices 2/3) recorded on the board + in the design doc: extractor item
(the perpetual birth/XP/item engine — the unbounded source), other rule-benders
(always-alive/dead, teleporter, time-phase), rune-shaped insertion/extraction,
and a maybe-easter-egg reverse rune generation. All fit the `Option<ItemType>`
cell model (modifiers ride on the item type), so they're additive.

**Not verified in-game.** To climb levels you need energy (foundry: Clicks →
Energy) feeding evolution. Caveat noted in the doc: level-up resets the cell grid
(colony lost) for now.

## Life: in-game bug fixes (timer + level-up grid wipe)

In-game testing surfaced: (2) Life evolves way faster than 1/s when fueled, and
(3) cells only flash by during execution. Both were the same root cause — a
**level-up cascade**: early XP thresholds are tiny (1,2,4,8…), so any activity
levels up repeatedly, and each level-up (a) reset the grid (cells vanish → flash)
and (b) reset the step timer (so it stepped immediately, not after 1s).

Fixes:
- **Step timer → tick cooldown.** Replaced `last_evolve_time: f32` (compared to
  `time.elapsed_secs()`) with `evolve_cooldown: u32` counted down per FixedUpdate
  tick (`EVOLVE_TICKS = 20` ≈ 1s at 20 Hz). A respawned/leveled-up life starts
  with a full cooldown, so it no longer fires instantly after respawn. `evolve`
  dropped its `Res<Time>` dep.
- **Level-up preserves the colony.** `levelup()` now copies the old cells into the
  larger new grid instead of starting empty — no more flashing, cells persist.

(1) Apple → greenish lump is *working as designed*: seeding always places Archaea
(the render-safe life form); the dropped item is consumed as nutrient. Noted in
the design doc + board as a possible future change (seed becomes the dropped item,
needs a renders-safely guard). 43 tests pass, clippy clean. Re-verify in-game.
