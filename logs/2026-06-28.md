# 2026-06-28

## Item model: drop `Terrain` structure, rename `Structure` → `BulkStructure`

Two review-driven changes to the physical taxonomy (see `references/item-model.md`):

- **Renamed `Structure` → `BulkStructure`.** "Structure" is too generic a name for
  what is specifically the matter-state of a *bulk* material.
- **Removed the `Terrain` variant.** Terrain isn't a state of matter — a terrain
  tile is just its underlying material (solid mud, liquid water). The land
  minigame already stored ingested solids/liquids as terrain cells; only the
  *default* terrain used `Terrain`. Now the default is `Item::solid(Mud, Lump)`,
  unifying it with ingested terrain, and "this cell is terrain" is purely the land
  minigame's concern (its `terrain` grid holds ordinary material items;
  `terrain_is_water` already keyed on the substance, not the structure).
  Dropped `Item::terrain`, `STRUCT_TERRAIN`, and the Terrain arms in
  draw/identifier/pack/unpack. `BulkStructure` is now `{ Gas, Liquid, Powder,
  Solid }`. (The Bulk/Discrete distinction's encoding was reworked immediately
  after — see the next entry.)

No save-format concern (pack/unpack isn't wired to disk yet, pre-release). 38
tests pass, clippy clean. Not re-verified in-game (land is still dormant; the
visible consumers — chest/ball_breaker/foundry/tree — are unaffected by the
terrain change, but a fresh in-game pass never hurts).
## Item model: nest the bit encoding to match the types

Follow-up to the BulkStructure rename: the packed encoding didn't mirror the Rust
types. The types nest (`PhysicalItem::Bulk(BulkItem { structure, .. })` vs
`PhysicalItem::Discrete`), but the bits flattened it — a single 5-bit `structure`
field held Gas/Liquid/Powder/Solid AND `Discrete` as a sentinel value (the doc's
original layout). Confusing.

Re-laid the physical payload to nest like the types:
- `kind` tag [60:59] = Bulk | Discrete (the `PhysicalItem` variant).
- For Bulk, the matter-state is a nested `BulkStructure` field [58:56].

With `Discrete` no longer occupying a slot in the structure field, `BulkStructure`
is now a clean 4-variant enum, so it also got the `IntEnum` treatment (Gas=0…
Solid=3) — `pack` = `as u64`, `unpack` = `try_from`. Removed the `STRUCT_*` consts
(replaced by `PHYS_KIND_BULK`/`PHYS_KIND_DISCRETE`). Updated the doc's layout
tables and property-query descriptions (`kind=Bulk`/`kind=Discrete`).

No save-format concern (not wired to disk yet). 38 tests pass (round-trips
confirm the nested encoding), clippy clean.
